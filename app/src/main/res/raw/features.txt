# Navigation & Interface
- Two-panel vertical layout (top + bottom)
  > Portrait: panels stacked vertically (top + bottom)
  > Landscape: panels arranged horizontally (left + right)
  > Tap anywhere in an inactive panel to activate it — file won't open, next tap works normally
- Draggable panel divider with 3 tap zones (bookmarks / 50-50 / tools)
  > Hold the divider and drag up/down (or left/right in landscape) to resize panels
  > Left zone (shows file count): tap → pie menu with 6 bookmark slots for quick folder access
  > Center zone: tap → reset panels to equal 50/50 split
  > Right zone (shows file count): tap → pie menu with tools (Trash, Analyzer, Duplicates, APK, Player, Reader)
  > While dragging files across the divider: left side = copy, right side = move
- Folder navigation with back button
  > ← button in panel header: go to previous folder from history, or parent folder if no history
  > → button: go forward in history
  > Both buttons gray out when navigation is not possible
  > Edge swipe can be configured as back in Settings → Gestures
- Clickable breadcrumbs for quick path navigation
  > Tap any folder segment in the path bar to jump directly to that level
  > Current folder (last segment) is not clickable
  > Path scrolls horizontally — auto-scrolls to show current folder
  > Right side shows folder size / storage total
- Navigation history per panel (← → like a browser)
  > Each panel keeps its own independent navigation history
  > Tap ← to go back, → to go forward — just like a browser
  > History works inside archives too
- Open current folder in other panel
  > Tap the overflow menu (⋮) in the panel header
  > Select "Open in other panel"
  > The same folder opens in the opposite panel — useful for comparing or moving files
- Active panel highlighted with border
  > Active panel: thicker border (2dp) in primary color + slightly elevated background
  > Inactive panel: thin border (1dp) in muted color
  > Switches automatically on tap, selection, or any action
- Scroll position preserved per panel
  > Each panel remembers its scroll position when you navigate away and return
- Desktop widget: favorites list on home screen, tap folder → app opens there
  > Long-press home screen → Add Widget → Haron Favorites
  > Shows list of your favorite folders with paths
  > Tap any folder → Haron opens directly in that folder
- Registered as system file handler: open from any app via "Open with", receive files via Share
  > Haron appears in system "Open with" chooser for files, archives, documents, media
  > Receive files via Share from any app
  > Auto-detects USB drives when connected

# Storage & Drives
- SD-card and external storage support
  > SD card appears in the Side Menu automatically when inserted
  > All file operations (copy, move, delete) work with SD cards just like internal storage
- All file operations work with content:// URI
  > Technical detail: on SD cards and external drives, Android uses SAF (Storage Access Framework)
  > Haron handles it transparently — no difference for you
- Access to Android/data and Android/obb via Shizuku (Android 11+)
  > Problem: Android 11+ blocks access to Android/data and Android/obb even with full storage permission
  > Solution: Shizuku — a free app that bypasses this restriction
  > 1. Install Shizuku from Play Store
  > 2. Start Shizuku via ADB or wireless debugging (instructions inside Shizuku)
  > 3. In Haron: Settings → Shizuku → Grant permission
  > After that: copy, move, delete, rename in Android/data and Android/obb work fully
- Auto-detect USB drives (OTG)
  > Plug in a USB drive via OTG adapter → Haron detects it automatically
  > USB drive appears in the Side Menu
- Navigate USB, safe eject
  > Work with files on the drive like a regular folder
  > Safe eject: tap "Eject" icon next to USB drive in Side Menu
- Free/total space indicator
  > Each storage shows how much space is used and total capacity

# Search
- Global search across entire device
  > How to open: long tap 🔍 in panel header, or voice "search" / "find"
  > Search screen: input field on top, filter chips below, results below that
- Two search modes: by name and by content
  > "Тт" button on the right — searches file names (always works, no indexing needed)
  > "📄" button — searches text INSIDE files (requires indexing, see below)
  > Switch between modes with one tap
- Search results
  > Each result: colored icon by file type, name, parent folder path, size, date
  > Tap result → open file
  > Long tap result → navigate to file location in explorer (shows the folder)

  > Results load in batches of 200 — scroll down for more
- Filters: category, size, date
  > Three rows of filter chips, scroll horizontally
  > Categories: All · Documents · Images · Audio · Video · Archives
  > Size: <1MB · 1-10MB · 10-100MB · 100M-1G · >1GB
  > Date: Today · Week · Month · Year · Older
  > Tap chip to enable, tap again to disable. Filters combine with each other
- Content search: text, DOCX, ODT, PDF
  > Switch to "📄" → type a word → finds it inside documents
  > Only works after indexing (🔄 button in header)
  > Without indexing, only file name search is available
- Audio/video metadata search
  > Finds by tags: artist, album, track title
  > Works after media indexing (🎵 button)
- ML Kit image labeling (objects on photos)
  > Type "dog" — finds photos with dogs
  > Works after visual indexing (📷 button)
- OCR on photos (text recognition)
  > Type text — finds photos where that text is visible (signs, screenshots, documents)
  > Works after visual indexing (📷 button)
- Archive search (file names inside)
  > Finds files inside ZIP/RAR/7z by name, without extracting
- Indexing: why and how
  > Without indexing, only file name search works — fast but limited
  > Indexing "teaches" the search engine about file contents — after that you can search by text, tags, objects on photos
  > Three buttons in search screen header — each handles its own type:
  > 🔄 Basic: reads text from documents, EXIF from photos, contents from archives
  > 🎵 Media: reads audio/video tags (duration, bitrate, codec, artist)
  > 📷 Visual: recognizes objects on photos (ML Kit) + text on photos (OCR)
  > Tap button → indexing starts with progress bar and current file name
  > First time: may take a few minutes (depends on file count)
  > Status below: "Indexed: X files · Y ago" + content index size
  > If some files couldn't be processed — "Skipped: N files" appears (tap to expand list)
- Background index update
  > Index updates automatically on file changes and screen on (every 30 min)
  > No need to re-run indexing manually each time

  > Tap 🔍 in panel header → quick filter files in current folder by name
  > "📄" button — switches to content search inside files (TXT, DOCX, ODT, PDF, etc.)
  > First content search indexes the folder (few seconds), subsequent searches are instant
  > Difference from global: local searches only current folder, global searches entire device
